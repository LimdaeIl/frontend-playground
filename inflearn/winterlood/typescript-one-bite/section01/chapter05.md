# 타입스크립트 핸드북 학습 정리

TypeScript는 JavaScript에 **정적 타입 시스템을 추가한 언어**다.
기존 JavaScript는 실행 시점에 타입 오류가 드러나는 반면, TypeScript는 **코드를 실행하기 전에 타입 오류를 잡을 수 있다**는 점이 가장 큰 차이이다.

이 특징 덕분에 코드의 안정성과 유지보수성이 크게 향상되며, 특히 **규모가 커질수록 효과가 극대화된다**.

---

핸드북은 단순 문법 나열이 아니라, **개념을 단계적으로 확장하는 구조**로 되어 있다.
처음에는 기본 타입과 사용법을 익히고, 이후에는 타입을 조합하거나 확장하는 방식으로 점점 복잡한 구조를 다루게 된다.

이 흐름을 이해하는 것이 중요하다. 단순 암기가 아니라 **타입을 어떻게 설계하는지**가 핵심이다.

---

기본적으로 TypeScript에서 가장 먼저 익혀야 할 것은 **기본 타입 시스템**이다.

```ts
let name: string = "kim";
let age: number = 20;
let isAdmin: boolean = false;
```

여기서 중요한 점은 단순히 타입을 붙이는 것이 아니라,
**값의 형태를 명확하게 정의한다는 것**이다.

이로 인해 잘못된 값이 들어오는 것을 컴파일 단계에서 차단할 수 있다.

---

하지만 모든 변수에 타입을 명시할 필요는 없다.
TypeScript는 **타입 추론(Type Inference)** 기능을 제공한다.

```ts
let name = "kim"; // 자동으로 string으로 추론
```

이 기능은 코드의 가독성을 높이고, 불필요한 반복을 줄여준다.
실무에서는 **명시 vs 추론을 적절히 섞는 것이 중요하다**.

---

여러 타입을 동시에 허용해야 하는 경우에는 **Union 타입**을 사용한다.

```ts
let id: string | number;
```

이 경우 `id`는 문자열 또는 숫자 둘 다 가능하다.

반대로 여러 타입을 합쳐 하나의 구조로 만들고 싶다면 **Intersection 타입**을 사용한다.

```ts
type User = { name: string } & { age: number };
```

이 개념은 단순 문법이 아니라,
**타입을 조합해서 더 복잡한 데이터 구조를 표현하는 핵심 도구**이다.

---

Union 타입을 사용하면 자연스럽게 등장하는 개념이 **Narrowing(타입 좁히기)**이다.

```ts
function print(value: string | number) {
  if (typeof value === "string") {
    console.log(value.toUpperCase());
  }
}
```

이 코드에서 TypeScript는 조건문을 통해
`value`가 string인지 number인지 **런타임 조건을 기반으로 판단**한다.

즉, Narrowing은
**“조건문을 통해 타입을 확정하는 과정”**이라고 이해하면 된다.

---

함수에서는 타입 정의가 더 중요해진다.

```ts
function add(a: number, b: number): number {
  return a + b;
}
```

여기서 핵심은 두 가지다.

- 매개변수 타입 정의
- 반환값 타입 정의

이를 통해 함수의 입력과 출력이 명확해지며,
잘못된 사용을 컴파일 단계에서 방지할 수 있다.

특히 반환 타입을 명시하는 습관은
**함수의 의도를 명확하게 드러내는 데 매우 중요하다**.

---

객체 구조를 정의할 때는 **interface 또는 type**을 사용한다.

```ts
interface User {
  name: string;
  age: number;
}
```

이 방식은 단순한 타입 선언이 아니라,
**데이터 구조를 설계하는 과정**이다.

특히 API 응답이나 데이터 모델을 정의할 때 매우 중요하며,
프로젝트 규모가 커질수록 필수적인 요소가 된다.

---

TypeScript에서 가장 강력한 기능 중 하나는 **Generics(제네릭)**이다.

```ts
function identity<T>(value: T): T {
  return value;
}
```

제네릭은 타입을 “고정”하지 않고,
**사용 시점에 결정되도록 만드는 방식**이다.

이로 인해 다음과 같은 장점이 생긴다.

- 코드 재사용성 증가
- 다양한 타입에 대응 가능
- 타입 안정성 유지

즉, 제네릭은
**유연성과 안정성을 동시에 확보하는 핵심 기능**이다.

---

기본 개념을 넘어서면, TypeScript는 타입을 “조작”하는 단계로 넘어간다.

대표적으로:

- `keyof`
- `typeof`
- mapped types
- conditional types

이들은 단순히 타입을 선언하는 것이 아니라,
**기존 타입을 기반으로 새로운 타입을 생성하는 도구**이다.

이 단계부터는 TypeScript가 단순 타입 체크 도구가 아니라
**타입 기반 프로그래밍 언어**라는 성격이 강해진다.

---

클래스 문법도 TypeScript에서는 타입과 결합된다.

```ts
class User {
  constructor(public name: string) {}
}
```

여기서 `public` 같은 접근제어자는
단순한 문법이 아니라 **데이터 접근 범위를 명확하게 제한하는 역할**을 한다.

즉, 객체지향 개념과 타입 시스템이 결합되어
더 구조적인 코드 작성이 가능해진다.

---

코드가 커지면 자연스럽게 파일을 분리해야 하고,
이때 사용하는 것이 **모듈 시스템**이다.

```ts
export function hello() {}
import { hello } from "./file";
```

이 구조를 통해 코드 의존성을 관리하고,
프로젝트를 체계적으로 구성할 수 있다.

---

전체 흐름을 보면 TypeScript의 핵심은 하나로 정리된다.

단순히 타입을 붙이는 것이 아니라,
**“타입을 통해 코드의 구조와 의도를 명확하게 표현하는 것”**이다.

이 관점이 잡히면 이후 학습 방향도 명확해진다.

- 타입은 데이터 구조를 정의하는 도구
- 조건문은 타입을 좁히는 도구
- 제네릭은 타입을 유연하게 만드는 도구

---

마지막으로 정리하면, TypeScript는 다음과 같은 가치를 제공한다.

- **정적 타입을 통한 안정성 확보**
- **코드 가독성과 의도 명확화**
- **대규모 프로젝트에서의 유지보수 용이성**
- **강력한 개발 도구 지원 (자동완성, 리팩토링 등)**

---

이 단계까지 이해했다면 다음 학습은 반드시 실습으로 이어져야 한다.

특히 다음을 연결하면 이해도가 급격히 올라간다.

- React에서 props 타입 정의
- API 응답 타입 모델링
- 실제 서비스 구조에 타입 적용

여기까지가 핸드북 초반부를 제대로 이해한 상태라고 보면 된다.
