# 타입스크립트의 동작 원리

타입스크립트의 동작 방식을 이해하려면 먼저
**대다수의 프로그래밍 언어가 어떻게 실행되는지**부터 이해해야 한다.

---

프로그래밍 언어는 기본적으로
**사람에게는 친숙하지만 컴퓨터에게는 친숙하지 않은 형태**이다.

컴퓨터는 다음과 같은 형태만 이해할 수 있다.

- 기계어
- 바이트 코드
- 이진수

반면 우리가 작성하는 코드는
영어 문법과 비슷한 **고수준 언어**이기 때문에
컴퓨터가 바로 이해하고 실행할 수 없다.

---

그래서 반드시 필요한 과정이 있다.

**컴파일(Compile)**

컴파일은
**사람이 작성한 코드를 컴퓨터가 이해할 수 있는 형태로 변환하는 과정**이다.

---

이 과정을 담당하는 것이 바로
**컴파일러**(Compiler)이다.

컴파일러는 우리가 작성한 코드를 받아서
최종적으로 **바이트 코드**(또는 **기계어**)로 변환한다.

---

여기서 중요한 점은
컴파일러가 코드를 바로 바이트 코드로 바꾸지 않는다는 것이다.

중간 단계가 하나 존재한다.

**AST (Abstract Syntax Tree, 추상 문법 트리)**

---

AST는 다음과 같은 특징을 가진다.

- 코드의 **구조만 남긴 트리 형태의 자료구조**
- 공백, 주석, 탭 등 **실행과 무관한 요소 제거**
- 코드의 의미 단위로 분해됨

---

예를 들어

```js
2 + (3 - 4);
```

이 코드는 AST로 변환되면

- `+` 연산이 루트
- 왼쪽: 2
- 오른쪽: `3 - 4`

이런 식의 **트리 구조**로 표현된다.

즉, AST는
**코드를 구조적으로 해석한 결과**라고 보면 된다.

---

이제 전체 컴파일 흐름을 보면 다음과 같다.

1. 코드 작성
2. AST로 변환
3. AST → 바이트 코드 변환
4. 컴퓨터 실행

---

여기까지가
**일반적인 프로그래밍 언어의 동작 방식**이다.

---

이제 타입스크립트의 동작 방식을 보면
여기서 한 단계가 추가된다.

---

타입스크립트도 동일하게 시작한다.

1. TypeScript 코드 작성
2. AST로 변환

여기까지는 완전히 동일하다.

---

하지만 그 다음 단계가 다르다.

**타입 검사 (Type Checking)**

---

컴파일러는 AST를 기반으로
코드에 타입 오류가 있는지 검사한다.

이 과정에서 두 가지 결과가 발생한다.

---

① 타입 검사 실패

- 타입 오류 존재
- 컴파일 중단
- JavaScript로 변환되지 않음
- 실행 불가능

예시:

```ts
let a: number = 1;
a.toUpperCase(); // 오류
```

number에는 toUpperCase가 없기 때문에 실패

---

② 타입 검사 성공

- 타입 오류 없음
- 다음 단계 진행

---

이후 타입스크립트는
다른 언어처럼 바이트 코드로 가지 않는다.

**JavaScript 코드로 변환**

---

즉, 전체 흐름은 다음과 같다.

1. TS 코드
2. AST 생성
3. 타입 검사
4. JS 코드로 변환
5. 실행 (브라우저 / Node.js)

---

여기서 매우 중요한 특징이 하나 있다.

**타입스크립트는 최종적으로 JavaScript를 생성한다**

---

그리고 이 JavaScript는
다시 일반적인 언어와 동일하게

- AST 변환
- 바이트 코드 변환
- 실행

과정을 거치게 된다.

---

또 하나 반드시 이해해야 하는 핵심 포인트가 있다.

**타입스크립트의 타입은 컴파일 이후 모두 사라진다**

---

예를 들어

```ts
let a: string = "1";
a.toUpperCase();
```

이 코드는 컴파일 후

```js
let a = "1";
a.toUpperCase();
```

로 변환된다.

즉,

- 타입 정보는 제거됨
- 실행에는 전혀 영향 없음

---

이 의미는 매우 중요하다.

타입스크립트는 실행을 위한 언어가 아니라 **검사를 위한 언어**라고 볼 수 있다

---

정리하면 타입스크립트의 역할은 다음과 같다.

- 코드 실행 전에 오류를 잡는다
- 안전한 JavaScript 코드로 변환한다
- 런타임 오류 가능성을 줄인다

---

마지막으로 전체 개념을 한 줄로 정리하면,

**TypeScript는 컴파일 과정에서 타입 검사를 수행하고,
문제가 없을 경우 안전한 JavaScript 코드로 변환하는 도구이다**

---

이 내용을 이해하면 이후 학습에서 중요한 포인트가 자연스럽게 연결된다.

- 왜 타입이 런타임에 존재하지 않는지
- 왜 JS 실행 환경이 그대로 사용되는지
- 왜 TypeScript는 “컴파일러”라고 불리는지
